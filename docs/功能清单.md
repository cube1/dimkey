## 怎么用这份清单

每条功能写成一个「用户问题」单元，其中第一行 **用户用自己的话提出的问题** 同时承担三种用途：

| 派生产物 | 它其实就是这份清单的…… |
|---|---|
| 操作手册 | 这些条目**按任务排序** + 配截图 |
| FAQ | 每条的「用户问题」**直接就是** FAQ 标题 |
| SEO | 每条的「用户问题」**就是一个搜索词**，可单独成页 |
| 官网落地页 | 从第二层「精准控制」里挑「隐藏王牌」提炼成卖点 |

写手册时三条原则：① 用「问题—解决」开头，别用功能名开头；② 每个功能标注操作入口（很多藏在右键菜单、列头、侧栏）；③ 第二层是差异化卖点，重点写、配场景。

## 核心心智模型：一致性的作用域 = 工作区

这是最值得单独讲清楚的概念，也是用户最容易困惑和**误判安全性**的地方。

脱敏时生成的「原文 ↔ 脱敏值」映射表绑定在**工作区**上，因此：

- **同一工作区内**：同一个原始值，无论出现在哪个文件、哪一行，脱敏结果都一致 → 数据关系不丢，也才能还原。
- **跨工作区**：每个工作区的映射、随机种子、序号计数各自独立 → 项目之间互相隔离。

但「跨工作区结果是否不同」取决于脱敏方式：

| 脱敏方式 | 同一工作区内 | 跨工作区 | 原理 |
|---|---|---|---|
| 掩码 Mask | 一致 | **也相同** | 按位遮盖，只看原值（138\*\*\*\*1234 到哪都一样） |
| 泛化 Generalize | 一致 | **也相同** | 固定规则，确定性 |
| 假数据 Fake | 一致 | **不同** | 每个工作区随机种子（`replace_seed`）不同 |
| 序号 Ordinal | 一致 | **不同** | 每个工作区独立计数（`replace_counters`） |

> ⚠️ **安全要点（必须写进手册）**：很多人以为「分到不同工作区 = 两批数据脱敏后不可关联」。这只在用 **假数据 / 序号** 时成立；如果用 **掩码**，即便分了工作区，相同原值的掩码结果仍然一样，**仍可能被交叉关联**。要做到「项目间不可关联」的隐私隔离，必须用生成式脱敏（假数据/序号）+ 分工作区。

## 功能主清单

### 第一层 · 快速上手（主线，人人都用）

**[Q] 怎么把文件导入进来开始脱敏？**
- 功能：`文件导入`
- 一句话：拖进来或点击选择，支持 xlsx / xls / csv / docx / pdf / txt 六种格式。
- 操作路径：主界面文件拖放区
- SEO：文档脱敏工具、Excel 脱敏、PDF 脱敏

**[Q] 没有文件，只想处理一段文字怎么办？**
- 功能：`粘贴文本`
- 一句话：直接粘贴一段文字进来脱敏，不用先存成文件。
- 操作路径：拖放区下方「粘贴文本」
- SEO：文本脱敏、在线脱敏一段话

**[Q] 文件有密码打不开怎么办？**
- 功能：`加密文件解锁`
- 一句话：受密码保护的 Excel / Word，输入密码即可解密导入。
- 操作路径：导入时自动弹出密码框
- SEO：加密 Excel 脱敏、带密码文档处理

**[Q] 它能自动认出哪些敏感信息？**
- 功能：`自动识别`
- 一句话：自动扫出手机号、身份证、银行卡、邮箱、地址、人名、公司名等多类信息（完整 21 类见附录 B）。
- 操作路径：导入后自动扫描
- SEO：身份证脱敏、手机号脱敏、敏感信息识别

**[Q] 认出来之后怎么选打码方式？**
- 功能：`脱敏策略`
- 一句话：三种方式——掩码（138\*\*\*\*1234）、假数据（换成逼真假值）、泛化（朝阳区→北京市）。
- 操作路径：右侧策略栏，每个类型可单独设
- SEO：数据脱敏方式、打码 vs 假数据

**[Q] 脱敏前能先看看效果吗？**
- 功能：`预览对比`
- 一句话：左右分屏看原文和脱敏后，敏感项高亮，确认无误再导出。
- 操作路径：识别完成后中间预览区
- SEO：脱敏预览、脱敏前对比

**[Q] 处理完怎么保存/导出？**
- 功能：`原格式导出`
- 一句话：按原来的格式导出，Excel 还是 Excel、PDF 还是 PDF，排版保留。
- 操作路径：结果页「导出」按钮选路径
- SEO：脱敏后导出、保留格式脱敏

### 第二层 · 精准控制（看家本领，最该被看见 → 落地页卖点池）

**[Q] 怎么让某个词不要被脱敏？**
- 功能：`关键字白名单`
- 一句话：把不想动的词加进白名单，识别时自动跳过。
- 操作路径：右键敏感项 →「加入白名单」，或策略栏白名单区手动加
- 场景：公司自家品牌名不算隐私，不想被打码。
- SEO：指定不脱敏、排除关键词

**[Q] 公司全称和缩写，能脱敏成同一个名字吗？**
- 功能：`别名组 / 关联脱敏`
- 一句话：把「阿里巴巴集团」「阿里」放进一个组，全文脱敏成同一个假名（系统自动取最长的作主名）。
- 操作路径：选中 2 个以上同类项 →「创建别名组」
- 场景：合同里全称简称混用，脱敏后要能看出是同一家。
- SEO：一致性脱敏、公司别名脱敏

**[Q] 同一个值出现多次，脱敏结果会一致吗？**
- 功能：`一致性替换`
- 一句话：**同一工作区内**，同一原始值脱敏成相同结果（默认开启）。一致性的作用域边界是工作区，详见上方「核心心智模型」。
- 操作路径：默认开启，无需设置
- 场景：一张表里同一个客户出现多行，脱敏后还得能对上是同一人。
- SEO：一致性替换、可关联脱敏

**[Q] 它认不出我行业的专有敏感词怎么办？**
- 功能：`自定义词典`
- 一句话：自己加敏感词，支持精确 / 模糊匹配，下次自动识别。
- 操作路径：设置 → 词典管理
- SEO：自定义敏感词、行业脱敏词库

**[Q] Excel 一整列都是身份证，要一个个点吗？**
- 功能：`列级脱敏`
- 一句话：自动推断每一列是什么类型，整列一键脱敏。
- 操作路径：预览页点列头 → 设置列规则
- 场景：几千行的客户表，按列设规则一次搞定。
- SEO：Excel 按列脱敏、整列打码

**[Q] 我想把某些词统一换成我指定的固定值，可以吗？**
- 功能：`模板替换模式`
- 一句话：不打码，而是按词典里设定的固定替换值整篇替换。
- 操作路径：工作区切换为「模板替换」模式
- SEO：批量替换词、文档套用模板

**[Q] 漏认 / 认错了能手动改吗？**
- 功能：`手动增删敏感项`
- 一句话：多认的右键删掉，漏认的框选文字手动标为敏感。
- 操作路径：预览 / 对比视图右键或框选
- SEO：手动标记脱敏、修正识别结果

**[Q] PDF 里想直接涂黑某一块怎么办？**
- 功能：`PDF 框选涂黑`
- 一句话：在 PDF 上拖框或选中文字，导出时对应区域涂黑。
- 操作路径：PDF 预览拖拽框选
- SEO：PDF 涂黑、PDF 黑条遮盖

### 第三层 · 可逆与批量（进阶效率）

**[Q] 脱敏后的文件还能还原回原文吗？**
- 功能：`还原`
- 一句话：凭脱敏时保存在**该工作区**的映射表逆向还原；映射表是工作区 / 任务级的，**跨工作区不能用别的映射还原**。
- 操作路径：还原页选历史任务 + 脱敏文件
- 场景：内部留底用脱敏版，需要时还能还原核对。
- SEO：脱敏可逆、脱敏还原

**[Q] 一次几十个文件要脱敏怎么办？**
- 功能：`批量处理`
- 一句话：一次导入多个文件，可逐个确认，也可统一配置全自动跑（带进度条）。
- 操作路径：导入多文件后选批量模式
- SEO：批量脱敏、文件夹批量打码

**[Q] 不同项目的规则能分开保存、重复用吗？**
- 功能：`工作区`
- 一句话：每个项目一个工作区，策略、词典、白名单、别名组、脱敏映射各自独立保存。
- 操作路径：左侧工作区列表新建 / 切换
- SEO：脱敏项目管理、脱敏规则复用

**[Q] 为什么同一个号码在两个项目里脱敏结果不一样？**
- 功能：`工作区隔离`
- 一句话：映射、随机种子、序号计数都按工作区独立。想让多批文件脱敏后能互相对应 → 放进同一工作区；想互相隔离 → 分到不同工作区（且用假数据 / 序号）。详见「核心心智模型」。
- 操作路径：通过新建 / 切换工作区控制
- SEO：跨文件一致脱敏、脱敏项目隔离

## 附录 A：脱敏方式说明

| 方式 | 说明 | 可调参数 | 样例 |
|---|---|---|---|
| 掩码 Mask | 部分遮盖 | 保留前 N 位 / 后 N 位 | 138\*\*\*\*1234 |
| 假数据 Fake | 生成逼真假值 | （随机种子，工作区级） | 张三 → 李雷 |
| 序号 Ordinal | 序号占位 | （计数器，工作区级） | 人员1、人员2 |
| 泛化 Generalize | 精确值改为范围 | 无 | 北京市朝阳区 → 北京市 |

（假数据 / 序号属「替换 Replace」策略的两种风格，泛化对部分类型如地址 / 职位 / 邮编默认启用。）

## 给作者的备注（不对外，写正式文档时处理）

1. **「纯本地 / 零网络通信」口径需校准**：当前已有 License 联网验证 + 可关闭的匿名统计两处联网。建议统一改成更准确也更站得住脚的说法：**「文件内容全程不上传，仅授权验证和（可关闭的）匿名统计会联网」**，避免被抓包质疑。
2. **识别类型 = SEO 金矿**：附录 B 的 21 类，每类都可单独成页做长尾 SEO。
3. **落地页对照**：仓库根目录有 `dimkey-landing-full.png`，可对照现有落地页文案，盘点哪些「隐藏王牌」（第二层）还没在官网露出。
4. **手册结构建议**：把「核心心智模型：一致性的作用域 = 工作区」单独成节——它既是核心特性，也是用户最容易困惑、最容易误判安全性的地方。
