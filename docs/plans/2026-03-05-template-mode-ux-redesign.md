# 模版替换模式 UX 重设计

**日期**: 2026-03-05
**状态**: 已定稿

## 背景

模版替换模式的实际用户场景与原始设计不一致。用户的典型场景是：手上有一份含假数据/代号占位符的模版文件（如"张三"代表真实客户名），希望批量替换成真实值。

### 当前痛点

1. **检测到的项不能直接设替换值** — 需要先加词典再设替换值，操作路径长
2. **词典条目没有编辑入口** — 只能添加/删除，不能修改已有条目
3. **左右不联动** — 左侧设替换值不同步右侧词典，反之亦然
4. **词典变更后左侧不更新** — 新增/修改词典条目后，对比视图不实时反映替换效果
5. **手动补充路径不顺** — 对于未检测到的占位符，添加替换映射的路径不够直接

## 设计方案：内联编辑优先

### 核心流程

```
导入模版文件 → 切换到「模版替换」模式
→ 系统自动检测（三层引擎照常运行）
→ 左侧对比视图显示高亮项
→ 用户操作（混合使用）：
   ├─ 点击已检测的高亮项 → Popover 输入替换值
   └─ 框选未检测到的文本 → Popover 输入替换值
→ 每次设置替换值 → 自动同步到右侧词典
→ 右侧词典也可直接编辑 → 实时反映到左侧
→ 满意后点击「导出」
```

### 1. Popover 交互改造

#### 已检测项 Popover（点击高亮项触发）

```
┌─────────────────────────┐
│  原文: 张三              │
│  类型: 人名              │
│  ─────────────────────  │
│  替换为: [输入框____]    │
│  ─────────────────────  │
│  [确认]  [取消]         │
└─────────────────────────┘
```

- 词典中已有替换值时，输入框预填充
- 确认后：更新/新增词典条目 → 左侧高亮变 teal → 右侧词典同步更新
- 已在词典中的项显示标识

#### 框选文本 Popover（选中文本后触发）

```
┌─────────────────────────┐
│  选中文本: XX有限公司     │
│  类型: [下拉选择____]    │
│  ─────────────────────  │
│  替换为: [输入框____]    │
│  ─────────────────────  │
│  [添加到词典并替换]      │
└─────────────────────────┘
```

- 需要选择敏感类型（非自动检测，需用户指定）
- 确认后：加入词典 + 左侧对应位置高亮 + teal 颜色

### 2. 右侧词典面板改造

#### 行内编辑

```
当前:  张三 → 李明  [x删除]
改后:  张三 → 李明  [✏️编辑] [x删除]

编辑模式:
  [张三____] → [李明____]  [✓保存] [✗取消]
```

#### 其他改进

- 模版模式下替换值标记为必填（引导用户填写）
- 映射总数统计：`词典映射 (3/5 已设替换值)`
- 词典变更即时生效，无需手动触发

### 3. 实时预览与联动机制

**核心改变**：模版替换模式下，预览不再依赖后端 `apply_desensitize`，而是前端直接根据词典映射实时渲染。

```
词典变更（任意来源）
    ├── 左侧 Popover 设置替换值
    ├── 右侧面板编辑词典条目
    └── 右侧面板添加/删除条目
        │
        ▼
  更新 Zustand Store 中的 dict_entries
        │
        ▼
  同步写入 Rust 后端（持久化）
        │
        ▼
  ComparisonView 重新渲染
    ├── 左侧：根据最新 dict_entries 更新高亮颜色
    │   - 有替换值 → teal 高亮，tooltip "原文 → 替换值"
    │   - 无替换值 → 灰色高亮
    │   - 手动添加的新项 → 也作为高亮项
    └── 右侧（预览）：直接用替换值渲染文本
```

**关键决策**：
- 预览是前端渲染，导出走后端
- 手动添加项：前端在文档内容中搜索该文本所有出现位置并高亮
- 高亮颜色方案不变：teal（有替换值）/ gray（无替换值）

### 4. 修改组件清单

| 组件 | 修改内容 |
|------|---------|
| `SensitivePopover/index.tsx` | 模版模式简化 UI，确认后同步词典，支持框选触发 |
| `StrategyPanel/DictSection.tsx` | 行内编辑功能，模版模式替换值必填引导 |
| `CenterPanel/ComparisonView.tsx` | 文本框选事件、手动项高亮、前端实时预览 |
| `HighlightedText/index.tsx` | 支持手动添加项的高亮 |
| `stores/workspaceStore.ts` | 词典 CRUD 操作方法，变更通知 |
| `hooks/useAutoDesensitize.ts` | 模版模式跳过自动 `apply_desensitize`，改前端预览 |

### 5. 不修改的部分

- Rust 后端 `apply_desensitize` 逻辑（导出时照常调用）
- 三层检测引擎（检测能力不变）
- 脱敏模式的所有逻辑
- 文件解析逻辑
- `DictEntry` 数据结构（已有 `replacement` 字段）

### 6. 边界约束

- 框选文本仅限纯文本内容（Excel 单元格内文本、Word 段落文本）
- 模版替换模式仍是 workspace 级别的模式切换
- 词典数据结构不变
