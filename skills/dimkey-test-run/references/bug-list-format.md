# Bug 清单格式

写入/更新 `e2e/bug-list.md`。

## 模板

```markdown
# Bug 清单

> 生成时间: {date}
> 来源: {执行范围描述}

## 活跃 Bug

| # | 优先级 | 模块 | 失败测试 | 根因描述 | 分类 |
|---|--------|------|----------|----------|------|
| BUG-001 | P0 | {模块} | `{test_name}` ({file}) | {根因} | {死循环/识别遗漏/逻辑错误/检测缺失/测试代码} |

## 环境问题（非 Bug）

| # | 说明 | 影响测试 | 解决方式 |
|---|------|----------|----------|
| ENV-001 | {说明} | {测试} | {解决方式} |

## 已关闭

| # | 关闭时间 | 模块 | 原失败测试 | 说明 |
|---|----------|------|------------|------|
| BUG-XXX | {date} | {模块} | `{test}` | 修复于 {commit/描述} |
```

## 规则

- 执行前先读取已有 bug-list.md，对比本次结果更新状态
- 本次通过的已有 Bug → 移入"已关闭"，标注关闭时间
- 仍失败的已有 Bug → 保留在"活跃"，更新描述（如根因变化）
- 新增失败 → 追加新编号，编号递增不复用
- 优先级：P0（阻塞/崩溃） > P1（功能缺陷） > P2（测试代码/边界）
- 环境问题单独归类，不占 Bug 编号
